网站（Web）

介绍

  • 发展

    • 静态网站
      使用简单的静态页面,没有很多的交互和数据传递。不能获取访问者数据，修改成本较高。

    • 页面 blingbling 的
      大量使用 Flash，页面动起来了，blingbling 的。

    • ASP,PHP,JSP 时代
      动态网站来了。网站上很多数据存放在服务器（的数据库）中，网站页面也可以由后端动态渲染。

    • AJAX 来了
      直到现在，web 服务器多半会返回整个页面或者文档，AJAX 的出现，让事情变得很有意思。AJAX 允许客户端的 JavaScript 脚本为局部页面提供请求服务，然后可以在无需回到服务器情况下动态刷新部分页面，无刷新更新数据这种模式异常流行，一直到现在。

    • MVC/MVVM
      后端和前端先后迎来 MVC/MVVM 架构。MVC Model-View-Controller。MVVM Model-View-ViewModel。很多新生的语言也开始流行，Python、Ruby 等满地开花。

    • 移动端浪潮
      Web 可以提供数据接口，供移动端应用调用。HTML5/CSS3 也使 Web App 流行。

    • Node.js
      Node 是一个基于 Chrome JavaScript 运行时建立的平台，使 JavaScript 第一次走出前端运行在了服务器上。同时 Node 也诞生了 npm，从此 JavaScript 也有了强大的包管理机制。

    • 未来…

  • 分类

    • 静态
      静态网站故名思议，所有页面都是静态的，一旦编码完成，除非进行源码的更改，否则页面永远就是这个样子。当然静态网站页面也是可以动的，比如插入 Flash。这种模式在 Web 发展初期很流行。一是那时候互联网技术出于起步阶段，没有技术条件；二是大家的审美和现在有很大的差距。
    • 动态
      后端脚本和各种后端框架的出现和逐渐流行，使 Web 技术有了长足的进步。这个时候，大家纷纷开始搭建动态网站。动态网站的好处显而易见。可以记录浏览者的信息、动态生成内容、更强大的内容管理等。
      后来，也出现了一些成熟的 CMS，如 WordPress、PHPCMS、Discuz等等。使建站成本和难度降低了很多。

    • 各有千秋
      这样看来，好像是动态网站完败静态网站一样。然而事实上并非如此。动态网站自然有自己压倒性的优势，然而随着 Web 越来越复杂，一些简单的展示性的内容并不需要后端来处理很多数据。于是，静态网站又回归了，特别是 Node 的流行，利用一些框架可以使用 Markdown 进行网页生成，一下子，静态网站的春来又来了。比如我的博客便是使用 Node 框架 Hexo 进行搭建的。使用 Hexo 你可以只关注于写作而不用考虑最终的生成效果，因为有大量的主题可供你使用，我会在后面介绍 Hexo 和 Markdown。

后端

  • 技术栈

    语言从来都不存在最好，脱离需求和应用环境谈优劣全是耍流氓。

    PHP —— 世界上最好的语言 Hypertext Preprocessor 。PHP 是一个非常简单的后端脚本语言，可能也是最为流行、使用范围最为广泛的后端脚本。
    优点是：成熟、稳定、中文资料、第三方框架和包很多。
    缺点是：没有逼格，效率和优雅度很成问题，只能用在 Web 编程。

    Python —— 一种简约语言，有着严格的缩进，通过缩进来代替大括号，使得代码很漂亮优雅。提倡的是 “There is only one way to do it” 的编程哲学。
    优点是：很优雅，不仅可以进行 Web 编程还可以开发游戏、客户端等，很适合做爬虫应用。
    缺点是： 中文资料匮乏，运行速度也比较慢。

    Ruby —— 一种魔幻语言，作者为日本人 松本行宏 。提倡 “There are many ways to do it” 的设计哲学，和 Python 正好背道而驰。
    优点是：使用灵活，语法糖很多，很 geek，rails 很典范。
    缺点是：中文资料匮乏，流行程度不如前者，第三方包不很丰富。

    一张三者对比图，不过是几年前的版本了。现在 PHP7 已经发布，Python 也已经 3.5，Ruby 已经 2.3 了。PHP-Python-Ruby 对比

    Node —— 后端新贵，使用 JavaScript 编写后端脚本。我会在 JavaScript 的现状与畅想 中详细介绍。

    假如你想帮他尽快找个活儿， 赚到钱， 推荐PHP。
    假如你想让他成为一个高效工程师， 推荐 Python。
    假如你想让他爱上他的工作， 推荐 Ruby。
    by 李天放 - 知乎

  • 如何学习？

    coding，coding and coding！重要的事情说三遍。

    学习一门编程语言或者框架，最快和最佳的方式莫过于不停的 阅读代码，练习代码。我推荐大家接触一门新语言/框架的时候先通读一下文档/语法，对这个语言/框架有一个整体的认识，然后开始练习。当然了，练习也是有技巧和选择性的。你可以阅读一些常用模块的源码，然后开始用自己的思路编写；也可以尝试用自己的思维解决一些身边的小问题。这些都是不错的方法。

    不管处于学习的哪个阶段，翻阅文档一定是不可或缺的环节。尽管很多官方文档比较意识流，并且可能没有中文译本。你也可以选择一些经过翻译或者重新整理后的文档。在编写代码的过程中，遇到不懂或者忘记的技术点，都可以翻阅文档。学习编程千万不能着急，不要想着一股脑掌握所有的语法点，慢工出细活，一步一步才能对技术有更深厚的掌握。

    这么讲，不是说基本功不重要，反而是基本功尤为重要。如果基本功不扎实的话，技术背后的原理就不太可能搞得清楚，会走很多弯路。我的建议是，在练习编码的同时，一定要经常性的阅读，阅读优秀的书籍和文章，阅读源码，阅读官方文档，这样可能就会事半功倍吧。

  • 框架

    框架固然好用，但是不要太依赖框架，而忘记了语言本身的魅力所在

    流行的后端框架会将常用的功能进行封装，更方便人们开发。同时，采用成熟的框架，安全性、性能等这些问题也并不需要考虑得太多，你可以只专注于编写代码，实现自己的想法。

    • PHP
      • ThinkPHP：一个国产的 MVC 开源 PHP 框架，是为了敏捷 WEB 应用开发和简化企业应用开发而诞生的。中文文档很全面，框架本身很成熟，使使用 PHP 开发大型应用更加方便。
      • Laravel：号称为 WEB 艺术家创造的 PHP 框架。有着优雅的语法，强大的工具集，是一个很现代化的 PHP 框架。
      • WordPress：全球最流行的博客系统。有丰富的主题风格，使用 WordPress 你可以秒搭博客。同时也可以将它作为一个内容管理系统（CMS）来使用。
      • yii，Discuz!，phpcms …
    • Python
      • Django：最流行的 Python Web 框架之一。
      • Flask：一个使用 Python 编写的轻量级 Web 应用框架。
      • Scrapy：用于抓取 Web 站点数据的爬虫框架。
      • web.py，web2py，pygame …
    • Ruby
      • Rails：高效的 Ruby Web 开发框架
    • Node
      • Express：Node 官方推荐的快速、开放、极简的 web 开发框架。
      • pomelo：网易开源的高性能、分布式游戏服务器框架。
      • ThinkJS：一款使用 ES6/7 特性全新开发的 Node.js MVC 框架，参考借鉴了 ThinkPHP 的特点。
      • hexo：静态博客
      • Ghost：Node 最强的博客平台

  • 如何学习？

    coding，coding and coding！重要的事情说三遍。

    同样的，对于前端而言，最好的学习方法也是不停的 阅读代码，练习代码。
    和后端相比，前端相对更为简单，有着更丰富的框架/库可以使用。对于 BootStrap，Amaze UI，Foundation，Semantic UI 等众多 CSS 框架，建议大家在快速建站的时候使用。前端工程师很容易陷在这些前端框架中，用习惯了框架，总会觉得所有的效果、布局自己都可以实现，殊不知，那些都是框架为你处理的，而不是你自己去实现的。在我看来这些前端框架实则是给后端工程师使用的，可以让不太了解前端的人也能做出漂亮出色的前端页面。由衷建议，使用这些 CSS 框架可以，但是不要忘记去理解背后的原理，尽早摆脱框架为好。

  • H5 ? HTML5 ？
    2014 年 10 月底，万维网联盟（W3C）宣布经过几乎8年的艰辛努力，HTML5 标准规范终于最终制定完成了，并已公开发布。在互联网上引起一番波澜。同时在之后的时间里，一大波打着 H5 旗号的页面风靡微信朋友圈、公众号。营销代有手段出,各领风骚数百天。要说现在哪些营销方式最能传播,屡屡刷爆朋友圈的 H5 页面肯定就是首当其冲的。加之易企秀，搜狐快站等 H5 页面制作工具的出现使其更加风靡。于是有了 元素必须飞起来，不飞不开心 的调侃。
    然而，本质上，这些页面的动画效果和 CSS3 关系更为紧密，和 HTML5 其实并无太大关联。硬说有的话，也就是加了背景音乐之类的吧。事实上，HTML5 在 W3C 发布标准之前早已被各大浏览器厂商支持了，只是在草案通过后，这种支持变得有依据，变得“合法化”而已。

  • Markdown

    Markdown 是一种轻量级的「标记语言」，它的优点很多，目前也被越来越多的写作爱好者，撰稿者广泛使用。看到这里请不要被「标记」、「语言」所迷惑，Markdown 的语法十分简单。常用的标记符号也不超过十个，这种相对于更为复杂的 HTML 标记语言来说，Markdown 可谓是十分轻量的，学习成本也不需要太多，且一旦熟悉这种语法规则，会有一劳永逸的效果。

    • 使用 Markdown 的优点
      • 专注你的文字内容而不是排版样式，安心写作。
      • 轻松的导出 HTML、PDF 和本身的 .md 文件。
      • 纯文本内容，兼容所有的文本编辑器与字处理软件。
      • 随时修改你的文章版本，不必像字处理软件生成若干文件版本导致混乱。
      • 可读、直观、学习成本低。
